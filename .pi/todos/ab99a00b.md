Make resuming an existing subagent session a first-class orchestration building block, rather than only a raw standalone tool action.

Typical use case example:
- A code-review subagent finishes with a request for the updated diff after fixes, or a child used `caller_ping` and should continue after receiving guidance.
- Instead of launching a fresh review/investigation from scratch, the orchestrator resumes the same prior session as part of the larger workflow.
- Downstream steps still receive a clean final result, but the resumed task preserves its accumulated conversation context.

Why this matters:
- Preserves session continuity, prior reasoning, and tool outputs.
- Avoids wasting work when a follow-up should continue an existing conversation.

Success criteria:
- Orchestration can represent "resume this existing session" as a normal step/task flow.
- Resume integrates cleanly with serial/parallel bookkeeping and final result aggregation.
