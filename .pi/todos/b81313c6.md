Migrate remaining skills and internal usage away from the old `subagent { chain / tasks }` style onto the current orchestration surface (`subagent`, `subagent_serial`, `subagent_parallel`).

Why this matters:
- Reduces conceptual duplication and maintenance burden.

- Helps the project converge on one mental model even if retirement work is deferred.

Constraints:
- Keep migration objective and incremental.
- Preserve behavior for existing skills during transition.
- Avoid mixing in unrelated retirement / cutover work in this todo.

Suggested steps:
- Inventory skills and prompts that still reference the old surface.
- Migrate highest-value / highest-frequency skills first.
- Update docs/examples/prompts together with code changes.
- Add or refresh tests where migration changes behavior.
