Implement async orchestration mode for `subagent_serial` and `subagent_parallel`, matching the spec's deferred `wait: false` behavior.

Why this matters:
- Highest-impact capability unlock among remaining out-of-scope items.
- Allows long-running orchestrations to continue in the background while the parent session keeps working.
- Especially valuable for `execute-plan`, long review/remediation loops, and multi-step verification workflows.

Scope ideas:
- Add a forward-compatible `wait` field to orchestration schemas.
- Define async return shape (e.g. orchestration id + pending task states) distinct from sync result shape.
- Add a compound completion watcher that reports aggregated results when the full serial/parallel run completes.
- Preserve cancellation / abort semantics and reporting.
- Ensure UI / steer behavior is coherent for background completion.

Success criteria:
- `subagent_serial({ wait: false, ... })` returns immediately and later reports final aggregated completion.
- `subagent_parallel({ wait: false, ... })` returns immediately and later reports final aggregated completion.
- Sync behavior remains unchanged when `wait` is omitted or `true`.
