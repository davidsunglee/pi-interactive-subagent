Add a direct way to focus a running pane-backed subagent from the widget and/or a dedicated `subagent_focus({ name })` tool.

Typical use case example:
- A user launches `subagent_parallel(...)` with 4-8 pane-backed tasks.
- One task looks stuck or especially interesting in the widget.
- Instead of remembering mux-specific navigation shortcuts, the user chooses that task from the widget (or calls `subagent_focus({ name })`) and jumps straight to the correct pane.

Why this matters:
- Speeds up the loop from overview -> inspect/intervene.
- Improves usability for users who are not fluent in tmux/zellij/wezterm navigation.

Success criteria:
- Users can reliably jump from orchestration UI/state to a named running pane.
- Behavior and backend limitations are documented clearly.
