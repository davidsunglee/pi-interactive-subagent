# Umbrella: Port selected upstream fixes from HazAT/pi-interactive-subagents v3.7.1

## Context

We compared this fork against upstream <https://github.com/HazAT/pi-interactive-subagents>.

- Fork point: `52cccd8` / upstream `v3.3.0`
- Upstream current at analysis time: `c0e4b35` / `v3.7.1`
- Current divergence: `181` local-only commits, `21` upstream-only commits (`git rev-list --left-right --count HEAD...hazat/main`)

Analysis artifacts:

- `docs/analysis/upstream-sync-analysis-pi.md`
- `docs/analysis/upstream-sync-analysis-claude.md`
- `docs/analysis/upstream-sync-ledger-pi.md`

Decision: **do not merge upstream wholesale**. This fork has substantial local headless execution, Claude support, orchestration tools, transcript/usage observability, and result-rendering changes. Treat upstream as a source of targeted patches.

## Porting strategy

- Cherry-pick where possible without excessive conflicts.
- Prefer manual reimplementation using upstream as reference when local headless/Claude/orchestration code diverges.
- Keep changes targeted; avoid adjacent refactors unless required by the port.
- Use TDD for implementation child todos.

## Verification policy

Implementation child todos should require:

- `npm run build`
- `npm test`
- full integration suite per the project test skill/process, including slow integration tests when available
