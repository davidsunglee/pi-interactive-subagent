# P1 Upstream sync: maintain v3.7.1 commit ledger and triage statuses

Parent: `TODO-356d8c41`

## Goal

Maintain a single source-of-truth ledger for every upstream commit added to `HazAT/pi-interactive-subagents` since our fork point.

Ledger path:

- `docs/analysis/upstream-sync-ledger-pi.md`

## Required status vocabulary

- `PROCESSED` — the behavior has been brought into our codebase or verified as already represented locally.
- `TO_BE_PROCESSED` — we have decided to bring the change in, but it has not been completed yet.
- `EXPLICITLY_OUT_OF_SCOPE` — we have decided not to bring the change in.
- `UNDECIDED` — no final call yet.
- `SUPERSEDED_OR_ALREADY_COVERED` — optional clarifying status for commits whose upstream change is unnecessary because the local fork already removed/covered the issue differently.

## Scope

- Catalog all 21 upstream-only commits from `52cccd8..hazat/main`.
- Track the target child todo/batch for each `TO_BE_PROCESSED` commit.
- Update commit statuses as the child todos land changes.
- Preserve rationale for out-of-scope and undecided commits.

## Acceptance criteria

- Ledger exists under `docs/analysis/`.
- Ledger includes every upstream-only commit since `v3.3.0` / `52cccd8`.
- Each commit has a status and rationale.
- Parent `TODO-356d8c41` links to the ledger and child todos.
