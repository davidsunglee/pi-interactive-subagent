## Post-focus-fix verification evidence

Added after the user-reported CMUX OS-focus regression where `cmux focus-pane` / `cmux focus-panel` could activate the CMUX macOS app while the user was working in another application.

Root cause:

- CMUX-internal focus restoration was issuing `cmux focus-pane` / `cmux focus-panel` unconditionally after launch.
- Those commands can bring the CMUX macOS app to the foreground when CMUX is backgrounded.
- The issue affected both the primitive `subagent` tool and orchestration tools because `subagent_run_serial` / `subagent_run_parallel` ultimately launch through the same pane surface primitive.

Fix:

- CMUX focus restoration now runs only when CMUX was the foreground macOS app at launch and is still foreground immediately before restore.
- Background CMUX launches skip `focus-pane` / `focus-panel`, so orchestration can open panes quietly without stealing OS focus.
- Foreground CMUX launches still restore CMUX-internal focus when launch side effects move focus to the new child/source/caller pane.
